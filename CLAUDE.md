# dai vault guide

## purpose
this vault is the knowledge and decision system for dai as a multi-tenant product factory.

## structure
- 01 Operating System = permanent founder rules and direction
- 02 Platform = reusable backend and agent architecture
- 03 Niches = domain-specific assembly lines
- 04 Products = productized outputs
- 05 Research = external findings and analysis
- 06 Execution = active planning and launch work

## niche doctrine
- generic agent roles stay stable across niches
- niches vary by data sources, prompts, thresholds, scoring rules, and output format
- do not hard-code niche assumptions into shared platform docs

## editing rules
- preserve markdown and obsidian-friendly structure
- do not rename top-level folders without asking
- when creating a new niche, use the niche template
- prefer practical v1 thinking over speculative scale
