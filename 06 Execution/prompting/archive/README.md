# Next-Slice Prompt Archive

**status:** archive of superseded next-slice proposals. **NOT current authority.**

Each file here is a prior `next-slice.md` proposal, archived by `dai-slice-prompt-architect` (MODE 5) before the
live `06 Execution/prompting/next-slice.md` was overwritten. Archived prompts preserve the proposal history so a
reviewer can see how next-slice thinking evolved.

## Rules

- **Do not execute an archived prompt.** It reflects the project state at its generation date and is almost
  certainly stale. The only live proposal is `../next-slice.md`, and even that is a proposal, not authority --
  `dai-slice-runner` validates live repo state before acting on it.
- **Naming:** archive as `next-slice-<YYYY-MM-DD>-<short-slug>.md` (the generated date + the recommended slice
  slug) so the timeline is legible.
- **Never edit an archived file** to "update" it; archives are immutable records. New thinking goes in a new
  `next-slice.md`, which is archived here in turn.
- **Provenance:** each archived file keeps its original header block (generated date, source handoff, expected repo
  heads, supersession note), which is exactly what marks it as historical.

This directory is governed by Slice Prompt Architecture Doctrine v1.
